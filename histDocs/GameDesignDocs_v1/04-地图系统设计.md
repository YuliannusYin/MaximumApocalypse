# 04 - 地图系统设计

> 本文件定义 PC 版地图系统的数据结构、25 种地块定义、任务卡内置布局+地块随机填充（D3）、拾荒系统、刷怪机制。
> 基于原版地块包文档（`桌游实体/基础版/地块包/MapBlocks.md`），应用所有设计决策（见 [00-总览与设计决策.md](00-总览与设计决策.md)）。
> 本文档为设计层面，不含代码。地块技能通过 SkillExecutor 调度执行（见 [02-角色系统设计.md](02-角色系统设计.md) 第 11 节）。

---

## 1. 概述

PC 版地图系统采用**任务卡内置布局 + 地块随机填充**（D3），从 25 种地块类型 / 38 个地块实例池中按任务卡内置二维数组填充布局。

核心特征：
- 25 种地块类型 + 38 个地块实例（见 D4）
- 4 方向相邻（北/南/东/西），无对角线
- 地块状态：未展示（背面）/ 已展示（正面）
- 玩家从面包车起始，逐步探索揭示地图
- 地块效果统一为 Skill（见 D24），由 SkillExecutor 调度

## 2. MapBlock 类定义

### 2.1 字段定义

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `block_id` | string | 地块类型唯一标识（文件名） | `"general_store"` |
| `block_name` | string | 地块显示名（中文） | `"百货商店"` |
| `scavenger_piles` | list[str] | 拾荒牌堆颜色列表（"red"/"blue"/"green"，空=不可拾荒） | `["green"]` |
| `monster_spawn_value` | int | 地块刷怪点数（见第 9 章） | 9 |
| `skills` | list[Skill] | 地块技能列表（见 D24，含展示/进入/行动/结束等触发技能） | 见第 5 章 |
| `is_starting` | bool | 是否起始地块（仅面包车为 true） | false |
| `is_special` | bool | 是否特殊地块（面包车=true） | false |
| `instances_count` | int | 该类型在地块池中的实例数（见第 6 章） | 1 |

### 2.2 实例字段（运行时）

每个地块实例（地块池中取出放置到地图上的地块）额外包含：

| 字段 | 类型 | 说明 |
|---|---|---|
| `instance_id` | string | 实例唯一标识（如 `"general_store_1"`） |
| `grid_pos` | Vector2i | 网格坐标 |
| `is_revealed` | bool | 是否已展示 |
| `monster_tokens` | int | 当前地块上的怪物标记数 |
| `scavenger_piles_override` | list[str] or null | 实例级覆盖 `scavenger_piles`（见 6.1 节城市街道3实例）；null 时使用类型默认值 |
| `scavenger_pile_remaining` | dict[str, int] | 各色拾荒牌堆剩余张数（运行时初始化） |

## 3. 地块触发时机扩展（TriggerTiming 增补）

02 文档 1.4 节定义的 TriggerTiming 枚举需追加以下地块专用时机：

```text
TriggerTiming (增补) =
    ON_REVEAL |              # 地块被展示时（翻面）
    ON_ENTER |               # 玩家进入地块时
    ON_LEAVE |               # 玩家离开地块时
    ON_ACTION |              # 玩家在地块上消耗行动执行地块行动
    ON_TURN_END_ON_BLOCK     # 玩家在此地块上结束回合时
```

| 触发时机 | 原版关键词 | 说明 |
|---|---|---|
| `ON_REVEAL` | 展示 | 地块从背面翻为正面时触发 |
| `ON_ENTER` | 进入 | 玩家移动进入已展示地块时触发 |
| `ON_LEAVE` | 离开 | 玩家从地块移动到相邻地块时触发 |
| `ON_ACTION` | 行动 | 玩家在地块上消耗 1 行动执行地块特殊行动（如机场、隧道） |
| `ON_TURN_END_ON_BLOCK` | 结束（在此地块上） | 玩家在此地块上结束回合时触发 |

> 完整 TriggerTiming 枚举合并：02 文档 1.4 节原 13 项 + 本节 5 项 = 18 项。

> 实现要点：MapBlock 实例化时，读取拾荒牌堆逻辑为 `scavenger_piles_override ?? scavenger_piles`（null 合并）。除城市街道外，其它地块实例的 `scavenger_piles_override` 均为 null。

## 7. 地图布局生成（D3）

### 7.1 决策概述（D3）

PC 版采用**任务卡内置布局 + 地块随机填充**策略：

- 任务卡 `map_layout` 字段存储二维数组，定义地图形状（固定排版）
- 地块内容从 38 个实例池随机抽取填充（保留重玩性）
- 程序化随机生成接口保留为未来扩展（玩家可选"内置布局"或"随机生成"）

### 7.2 TaskCard.map_layout 字段定义

| 字段 | 类型 | 说明 |
|---|---|---|
| `map_layout` | list[list[int]] | 二维数组，定义地图形状（固定排版） |
| `required_blocks` | list[string] | 必需地块类型列表（按顺序填充到占位值为 `3` 的格子） |

`map_layout` 数组值含义：

| 值 | 含义 | 填充逻辑 |
|---|---|---|
| `0` | 空位 | 不放置地块 |
| `1` | 任意地块 | 从剩余实例池随机抽取一个实例 |
| `2` | 面包车 | 从实例池取 `van_1` 实例，标记 `is_revealed=true` |
| `3` | 必需地块占位 | 从 `required_blocks` 队首取一个类型，从该类型实例池随机选一个实例 |

> TaskCard 完整字段定义见 08-任务系统设计.md。`map_layout` 数组大小由任务卡决定（不固定 11x11）。

### 7.3 填充算法步骤

```text
输入：task.map_layout, task.required_blocks, seed
输出：grid (grid_pos → block_instance)

1. 初始化：
   - instance_pool ← 38 个地块实例（深拷贝，见第 6 章）
   - required_queue ← task.required_blocks 转为队列
   - grid ← 空网格

2. 遍历 map_layout（按行优先）：
   for each (row, col, value) in map_layout:
     pos = (row, col)
     if value == 0: continue  # 空位
     elif value == 2:  # 面包车
       van = instance_pool.find(block_id="van")
       grid[pos] = van (is_revealed=true)
       instance_pool.remove(van)
     elif value == 3:  # 必需地块占位
       block_type = required_queue.dequeue()
       candidates = instance_pool.filter(block_id=block_type)
       if candidates.empty: error("必需地块实例不足")
       instance = random.choice(candidates)  # 类型内随机选实例
       grid[pos] = instance (is_revealed=false)
       instance_pool.remove(instance)
     elif value == 1:  # 任意地块
       instance = random.choice(instance_pool)
       grid[pos] = instance (is_revealed=false)
       instance_pool.remove(instance)

3. 验证：
   - required_queue 为空？（所有必需地块已放置）
   - 地块池实例数足够？（非空格子数 ≤ 38）

4. 返回 grid
```

### 7.4 实例选择策略

- **类型内随机选实例**：同类型多实例地块（如3个城市街道）从该类型所有剩余实例中随机选一个
- 优势：保留实例差异（不同颜色/刷怪值），同时保留重玩性
- 示例：填充"必需地块=城市街道"占位时，从3个城市街道实例中随机选1个

### 7.5 玩家移动规则

| 行动 | 消耗 | 触发 |
|---|---|---|
| 移动到相邻已展示地块 | 1 行动 | ON_LEAVE（离开原地块）+ ON_ENTER（进入新地块） |
| 展示相邻未展示地块 + 移动进入 | 1 行动 | ON_REVEAL（翻面）+ ON_ENTER |
| 在地块上执行地块行动（如机场/隧道） | 1 行动 | ON_ACTION |

### 7.6 地图边界与可达性

- 地图形状由 `map_layout` 二维数组大小决定（不固定 11x11）
- 数组边界外无地块
- 所有非空地块从面包车可达（任务卡设计时需保证连通性）
- 若任务卡布局存在孤岛地块 → 算法验证时报错，需修正任务卡

### 7.7 程序化随机生成接口（未来扩展）

保留接口 signature：

```text
generate_map(map_size, required_blocks, min_distance_to_van, seed) → grid
```

未来实现思路：
- 玩家可选"内置布局"（使用任务卡 `map_layout`）或"随机生成"（调用此接口）
- 算法：BFS 从面包车扩展，按任务约束加权随机选地块
- MVP 阶段不实现，仅保留接口签名

## 8. 地块拾荒系统

### 8.1 拾荒牌堆初始化

- 每色拾荒牌堆独立洗混（详见 05-卡牌系统设计.md）
- 地块实例初始化时按 `scavenger_piles` 关联对应色堆引用
- 当某色堆被抓空 → 该色无法拾荒（见 D12），地块拾荒标记 UI 变灰

### 8.2 拾荒行动定义

- **标准拾荒行动**：玩家消耗 1 行动点，从当前地块任一未抓空的色堆抓 1 张拾荒卡到手牌
- **免费拾荒行动**（effect_id=`free_scavenge_once`）：不消耗行动点，从当前地块任一未抓空的色堆抓 1 张拾荒卡到手牌
- 多色地块：玩家弹窗选择 1 个未抓空的色堆

### 8.3 拾荒卡进入手牌

按 D9：拾荒到的装备卡先进**手牌**（占手牌位），后续需消耗 1 行动"使用"才能装备到装备区。例外：燃料卡（D5）抓取时立即三选一，不进手牌。

## 9. 地块刷怪点数（怪物出生阶段）

### 9.1 字段含义（已确认）

`monster_spawn_value` = **骰子点数匹配值**（范围 0-12）。

在每个玩家回合的**"怪物出生"阶段**：

```text
1. 玩家投掷 2 颗大骰子
2. 记两骰点数之和为 X（范围 2-12）
3. 在每个正面朝上（is_revealed=true）且 monster_spawn_value == X 的地块上
   放置 1 个怪物标记
```

特殊值：
- `0`（军事基地）：永不通过骰子机制放置怪物标记（安全地块）
- 数值 2-12 对应 2 颗大骰子点数和的取值范围

### 9.2 怪物标记来源

- **主要来源**：怪物出生阶段骰子匹配（见 9.1）
- **次要来源**：
  - 工厂 ON_REVEAL：向相邻地块各加 1 标记
  - 部分卡牌效果可手动添加/移除地块标记（如钉子炸弹、无人机攻击）

### 9.3 怪物标记 vs 怪物卡

| 维度 | 怪物标记（Monster Token） | 怪物卡（Monster Card） |
|---|---|---|
| 来源 | 地块出生阶段骰子匹配 / 卡牌效果 | 玩家抓怪物卡（如城市街道 ON_ENTER） |
| 位置 | 地块上（不纠缠玩家） | 玩家面前（纠缠玩家，见 R3） |
| 攻击 | 不主动攻击 | 在玩家回合结束自动攻击纠缠玩家 |
| 处理 | 卡牌效果可移除（如无人机攻击） | 玩家攻击消灭 / 卡牌效果赶走 |

### 9.4 怪物标记上限（D16）

- 全局怪物标记总数 ≥ 上限（默认 30）→ 玩家失败
- 上限由初始化程序返回值决定，可由任务/难度配置覆盖
- UI：地图边缘显示当前标记数 / 上限

### 9.5 待补充：骰子机制

- 是否存在"小骰子"？原版若有多套骰子，需在 09-术语表 中统一

## 10. 地块技能与 SkillExecutor 集成

### 10.1 SkillExecutor 扫描范围扩展

02 文档 11.2 节执行流程中，SkillExecutor 扫描对象需包含：

```text
扫描对象 = [
    当前回合玩家,
    玩家装备区所有装备,
    玩家当前所在地块（MapBlock 实例）,         # ← 地块技能
    玩家所在地块上的所有怪物标记,
    玩家面前所有纠缠怪物,
]
```

### 10.2 地块事件触发流程

```text
1. 玩家执行"展示相邻地块"操作
   ↓
2. SkillExecutor 发出 ON_REVEAL 事件，target = 被展示的地块实例
   ↓
3. SkillExecutor 扫描该地块的 skills 列表
   ↓
4. 对每个 trigger_timing == ON_REVEAL 的 Skill：
   - ConditionChecker 检查 condition
   - 通过 → 调用 EffectHandler[effect_id]
   ↓
5. 玩家移动进入该地块
   ↓
6. SkillExecutor 发出 ON_ENTER 事件 → 同流程触发 ON_ENTER 技能
```

### 10.3 ON_LEAVE 时序问题

森林"穿越抓怪"（`forest_pass_through`，ON_LEAVE）的特殊时序：

```text
玩家从森林 A 移动到地块 B
  ↓
1. ON_LEAVE 事件触发（target=森林 A）
   → 触发 forest_pass_through，但需检查 condition："本回合进入过森林 A"
   → 满足 → draw_monster_1
2. ON_ENTER 事件触发（target=地块 B）
   → 触发地块 B 的 ON_ENTER 技能
```

> ConditionChecker 需记录玩家本回合的"进入历史"列表 `entered_blocks_this_turn: list[block_id]`。
