# 08 - 任务系统设计

> 本文件定义 PC 版任务系统的 TaskCard 数据结构、13 个任务设计、胜利条件、燃料机制、首领卡机制。
> 基于原版任务包文档（`桌游实体/基础版/任务包/basic-mission_0.md` ~ `_12.md`），应用所有设计决策（见 [00-总览与设计决策.md](00-总览与设计决策.md)）。
> 本文档为设计层面，不含代码。地图布局生成见 [04-地图系统设计.md](04-地图系统设计.md) 第 7 章（`map_layout` 字段）。

---

## 1. 概述

PC 版任务系统包含 13 个任务（任务 0-12），按 4 个叙事章节分组：

| 章节 | 任务编号 | 怪物包 | 主题 |
|---|---|---|---|
| 第一章 僵尸篇 | 0、1、2、3 | zombie（僵尸） | 僵尸末日、解药研制 |
| 第二章 突变体篇 | 4、5、6 | mutant（突变体） | 核冬天、拆弹、核辐射 |
| 第三章 外星人篇 | 7、8、9 | alien（外星人） | 侦查、情报、反击 |
| 第四章 机器人篇 | 10、11、12 | robot（机器人） | 运输、防御、反攻 |

核心特征：
- 每个任务卡内置 `map_layout` 二维数组定义地图形状（见 D3）
- 任务卡配置拾荒牌堆、怪物包、必需地块、首领卡机制
- 难度由特别简单递进到非常困难

## 2. TaskCard 数据结构

### 2.1 字段定义

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `task_id` | string | 任务唯一标识 | `"mission_0"` |
| `task_name` | string | 任务名（中文） | `"教程"` |
| `difficulty` | Difficulty | 难度枚举（见 2.3） | `VERY_EASY` |
| `monster_pack` | MonsterPack | 怪物包类型（见 2.4） | `ZOMBIE` |
| `intro` | string | 任务介绍/背景 | `"购物中心避难..."` |
| `objective` | string | 胜利条件描述 | `"找到4桶燃料..."` |
| `fuel_required` | int or null | 启动面包车所需燃料（null=不需燃料，见 D18） | 4 |
| `map_layout` | list[list[int]] | 二维数组定义地图形状（见 04 文档 7.2 节） | 见 2.2 节示例 |
| `required_blocks` | list[string] | 必需地块类型列表（按顺序填充到占位值 3 的格子） | `["mall"]` |
| `starting_block` | string or null | 起始地块类型（null=默认面包车，见 D3 Q6） | `"mall"` |
| `scavenger_decks` | dict[DeckColor, dict[str, int]] | 三色拾荒牌堆配置（key=牌堆标识，value={card_id: count}，见第 5 章） | 见第 5 章 |
| `boss_config` | BossConfig | 首领卡配置（见 6.2 节） | 见 6.2 节 |
| `target_markers` | list[TargetMarkerConfig] | 目标标记配置（最多 3 个） | 见第 7 章 |
| `monster_token_limit` | int | 怪物标记上限（默认 30，见 D16） | 30 |
| `special_rules` | list[string] | 任务特殊规则描述 | `["核辐射扩散..."]` |

### 2.2 map_layout 字段说明

`map_layout` 二维数组定义地图形状，值含义见 [04-地图系统设计.md](04-地图系统设计.md) 第 7.2 节：

| 值 | 含义 | 填充逻辑 |
|---|---|---|
| `0` | 空位 | 不放置地块 |
| `1` | 任意地块 | 从剩余实例池随机抽取 |
| `2` | 面包车 | 取 van_1，is_revealed=true |
| `3` | 必需地块占位 | 从 required_blocks 队首取类型，填充实例 |

**起始地块展示状态**：填充算法中，若 `required_blocks` 中某类型的地块 == `starting_block`，则该地块 `is_revealed=true`（如任务 0 的购物中心）。

**任务 0 map_layout 示例**（基于原版网格示意）：

```text
原版网格：
|购物中心|未知地块|未知地块|未知地块|未知地块|无地块|
|未知地块|未知地块|未知地块|无地块|未知地块|面包车|
|无地块|未知地块|未知地块|未知地块|未知地块|未知地块|

PC 版 map_layout：
[
  [3, 1, 1, 1, 1, 0],
  [1, 1, 1, 0, 1, 2],
  [0, 1, 1, 1, 1, 1]
]
required_blocks = ["mall"]
starting_block = "mall"  # 玩家从购物中心开始
```

> 其他任务（1-12）的 `map_layout` 具体值待数据录入阶段根据原版地块清单设计。设计原则：保证所有非空地块从起始地块可达（连通性）。

### 2.3 Difficulty 枚举

```text
Difficulty = TUTORIAL | VERY_EASY | EASY | NORMAL | HARD | VERY_HARD
```

| 枚举 | 中文 | 任务 |


## 4. 任务特殊规则

### 任务 0 - 教程
- 购物中心初始时已展示（`is_revealed=true`）
- 地图布局为固定网格（原版提供了网格示意，见 2.2 节）
- 胜利：找到 4 桶燃料，所有人上车且有足够燃料

### 任务 1 - 解救科学家
- 在警察局可花费 2 行动解救科学家
- 到达警察局的第一名玩家抓取首领卡
- 科学家和首领卡初始放到一边
- 胜利：解救科学家并带回面包车逃离

### 任务 2 - 收集样本
- 首领设置：从怪物牌库抓 9 张牌 + 首领卡正面朝下洗混放牌库底
- 到达警察局的第一名玩家抓首领卡
- 胜利：杀死 8 个僵尸（步行者/喷吐者/狗/士兵各 2 个）收集样本，带燃料回面包车

### 任务 3 - 研制解药
- **不需燃料**（D18）：终点为医院，不依赖面包车
- 科学家装备在一名求生者面前
- 同地块可像交易拾荒牌一样交易科学家
- 首领设置：9 张牌 + 首领卡洗混放牌库底
- 胜利：收集 2 医疗用品 + 3 解毒剂，带科学家到医院且回合结束无死亡

### 任务 4 - 核冬天
- **不需燃料**（D18）：终点为避难所
- 胜利条件：收集 3 燃料 + 2 脏毯子 + 2 老报纸 + 食物增加到 5，带物品返回避难所并摧毁所有尾随怪物
- 3 燃料是任务交付物（见 R4），不用于启动面包车
- 首领设置：9 张牌 + 首领卡洗混放牌库底

### 任务 5 - 拆除炸弹
- **回合限制**（D20）：解除炸弹后只有 3 个回合返回面包车
- 找到日记本 → 前往电厂 → 花 2 行动拆弹 → 3 回合内回面包车
- 目标标记必须距面包车至少 5 个地块
- 到达目标地块获得日记本并抓首领卡
- PC 版加回合数倒计时 UI 提示

### 任务 6 - 核辐射
- 面包车放地图最中间，从面包车向外构建地图
- **核辐射扩散机制**：怪物出生阶段投出 7 时，展示地图最外围的一个地块；所有外围地块展示后再次投出 7 时，移除一个外围地块
- 收集 3 个多余配件修理面包车（每次 1 张修 1 次，共修 3 次）
- 首领设置：5 张牌 + 首领卡洗混放牌库底

### 任务 7 - 侦查外星人地区
- **全图探索要求**：必须展示所有地图块
- 首领设置：选择一个地块放目标标记，展示该地块的玩家抓首领牌
- 胜利：展示所有地图块，带足够燃料返回面包车

### 任务 8 - 情报恢复
- **不需燃料**（D18）：终点为军事基地（初始正面向上展示）
- **潜行检定机制**：到达目标地块时投潜行检定
  - 成功：花费 1 行动解救科学家并装备
  - 失败 + 有日记本：记下信息
  - 失败 + 无日记本：**游戏失败**
- 目标标记距军事基地至少 5 地块
- 第一个到达目标地块的玩家抓首领卡

### 任务 9 - 人类反击
- **不需燃料**（D18）：终点为现场（外星飞船）
- 2 个被守卫的外星发射器地块（各 1 目标标记 + 3 怪物标记）
- 花费 1 行动摧毁 1 个发射器
- 首领：把 2 张首领卡洗入怪物牌库
- 胜利：摧毁 2 个发射器，把科学家带到现场上传病毒

### 任务 10 - 运输
- **最高燃料需求**：6 桶（基础版最高）
- 面包车与军事基地分别放在地图相对两边
- 3 个目标标记地块（彼此距离至少 2 地块）
- **多目标标记效果机制**：
  - 标记 1：首个到达者获得 3 多余零件 + 2 医疗用品
  - 标记 2：首个到达者抓首领卡 + 获得 3 脏毯子
  - 标记 3：首个到达者获得 2 燃料 + 1 医疗用品（不从拾荒牌库拿取）
- 胜利：带 6 燃料 + 3 脏毯子 + 2 医疗用品 + 2 多余零件返回基地

### 任务 11 - 保护基地
- **不需燃料**（D18）：起点和终点都是军事基地
- **起始异常**：玩家在军事基地开始，任务开始时不抓取怪物卡
- 3 个目标标记地块（彼此距离至少 3 地块且距基地至少 3 地块）
- 每个目标标记地块放 3 个怪物标记
- 目标标记所在地块所有怪物标记被清除后移除该目标标记
- 首领卡洗入怪物牌库下半部
- 胜利：清除 3 个目标标记，返回军事基地并杀死所有玩家面前的怪物卡

### 任务 12 - 烧死那群机器人
- **地块摧毁机制**：被标记地块被玩家进入后，任何玩家可花费 1 行动 `remove()` 该地块和上面的一切
- 3 个目标标记地块（彼此距离至少 3 地块）
- 首领卡随机洗入怪物牌库 + 牌库底额外放 1 张首领卡（共 2 张首领卡）
- 胜利：摧毁 3 个标记地块，携带足够燃料返回面包车

## 5. 拾荒牌堆配置

每个任务通过 `scavenger_decks` 字段配置三色拾荒牌堆的组成。**每个拾荒牌堆是混合牌堆**，可包含红/蓝/绿/灰 4 色大类的任意卡牌（卡牌大类颜色与牌堆标识是独立概念，详见 [05-卡牌系统设计.md](05-卡牌系统设计.md) 第 3.1 节）。

### 5.1 数据结构

```text
scavenger_decks: dict[DeckColor, dict[str, int]]
# key   = 牌堆标识（"red"/"blue"/"green"，DeckColor 枚举，见 05 文档 4.3 节）
# value = {card_id: count} 字典，记录该牌堆中各卡的副本数
#       - card_id 引用 05 文档第 3 章定义的 25 种卡牌之一
#       - count 表示该卡在此牌堆中的张数
```

### 5.2 任务 0 配置示例

来自原版 `basic-mission_0.md`，展示混合牌堆特征（4 色大类卡共存于同一牌堆）：

```text
scavenger_decks = {
    "red": {                          # 红色拾荒牌堆（共 12 张）
        "food_1": 2,                  # 绿色大类卡
        "fuel": 4,                    # 红色大类卡
        "med_supply_small": 2,        # 红色大类卡
        "ammo_2": 1,                  # 蓝色大类卡
        "binoculars": 1,              # 蓝色大类卡
        "spare_parts": 1,             # 灰色大类卡
        "nothing": 1                  # 灰色大类卡
    },
    "green": {                        # 绿色拾荒牌堆（共 12 张）
        "food_1": 6,                  # 绿色大类卡
        "fuel": 2,                    # 红色大类卡
        "ammo_2": 1,                  # 蓝色大类卡
        "flashlight": 1,              # 蓝色大类卡
        "spare_parts": 1,             # 灰色大类卡
        "nothing": 1                  # 灰色大类卡
    },
    "blue": {                         # 蓝色拾荒牌堆（共 12 张）
        "food_1": 2,                  # 绿色大类卡
        "fuel": 2,                    # 红色大类卡
        "ammo_2": 3,                  # 蓝色大类卡
        "binoculars": 1,              # 蓝色大类卡
        "pistol": 1,                  # 蓝色大类卡
        "walkie_talkie": 1,           # 蓝色大类卡
        "spare_parts": 1,             # 灰色大类卡
        "nothing": 1                  # 灰色大类卡
    }
}
```

> 示例中每个牌堆都包含红/蓝/绿/灰 4 色大类的卡牌。例如"红色牌堆"里有绿色食物卡、蓝色弹药卡、灰色事件卡。这证实卡牌大类颜色 ≠ 牌堆标识。

### 5.3 拾荒牌堆组成规律（按章节）

| 章节 | 红色堆特点 | 绿色堆特点 | 蓝色堆特点 |
|---|---|---|---|
| 僵尸篇 (0-3) | 基础组成 | 基础组成 | 基础组成 |
| 僵尸篇 (1-3) | 加入解毒剂 | 加入伏击！ | 加入解毒剂 |
| 突变体篇 (4-6) | 加入背包、脏毯子、老报纸 | 基础组成 | 基础组成 |
| 外星人篇 (7-9) | 弹药增加 | 加入日记本 | 基础组成 |
| 机器人篇 (10-12) | 医疗用品较多 | 基础组成 | 基础组成 |

> 各任务的具体拾荒牌堆清单（含副本数）待数据录入阶段从原版任务文件提取。

## 6. 燃料与首领卡机制

### 6.1 燃料机制汇总（D18）

| 燃料需求 | 任务 | 说明 |
|---|---|---|
| 需要燃料逃离面包车 | 0(4)、1(4)、2(4)、5(3)、6(3)、7(4)、10(6)、12(3) | 胜利条件含"开车逃离" |
| 不需燃料 | 3、4、8、9、11 | 胜利条件为到达特定地点（D18） |

任务 4 的 3 燃料是任务交付物（见 R4），不用于启动面包车。

### 6.2 首领卡机制（D19）

`boss_config` 字段定义首领卡处理方式：

```text
BossConfig {
    mechanic: BossMechanic    # 首领卡机制类型
    count: int                # 首领卡数量（默认 1）
    setup_cards: int         # 洗入牌库底前抓取的牌数（如 9 张）
    trigger_block: string or null  # 触发地块（如 "police_station"）
}
```

BossMechanic 枚举与各任务配置：

| 任务 | 机制 | 配置 |
|---|---|---|
| 0 | `NONE` | 教程无首领 |
| 1 | `TRIGGER_ON_BLOCK` | 到达警察局的首名玩家抓首领卡 |
| 2 | `SHUFFLE_BOTTOM` | 9 张牌 + 首领卡洗混放牌库底；到达警察局首名玩家抓首领卡 |
| 3 | `SHUFFLE_BOTTOM` | 9 张牌 + 首领卡洗混放牌库底 |
| 4 | `SHUFFLE_BOTTOM` | 9 张牌 + 首领卡洗混放牌库底 |
| 5 | `TRIGGER_ON_BLOCK` | 到达目标地块首名玩家抓首领卡 |
| 6 | `SHUFFLE_BOTTOM` | 5 张牌 + 首领卡洗混放牌库底 |
| 7 | `TRIGGER_ON_REVEAL` | 展示目标标记地块的玩家抓首领牌 |
| 8 | `TRIGGER_ON_BLOCK` | 第一个到达目标地块的玩家抓首领卡 |
| 9 | `SHUFFLE_INTO_DECK` | 2 张首领卡洗入怪物牌库 |
| 10 | `TRIGGER_ON_MARKER` | 标记 2 触发首领 |
| 11 | `SHUFFLE_LOWER_HALF` | 首领卡洗入怪物牌库下半部 |
| 12 | `SHUFFLE_PLUS_BOTTOM` | 随机洗入 + 牌库底额外 1 张（共 2 张） |

> 任务 2（D19）：修正原版矛盾——首领卡洗入怪物牌库底 10 张中，到达警察局时触发额外效果。

## 7. 目标标记机制

游戏配件共 3 个目标标记，通过 `target_markers` 字段配置：

```text
TargetMarkerConfig {
    position_constraint: MarkerConstraint  # 位置约束
    initial_monster_tokens: int            # 初始怪物标记数（如任务 9/11 的 3 个）
    on_first_arrival: EffectId             # 首个到达者的效果（如任务 10 的标记效果）
}
```

目标标记使用情况：

| 任务 | 目标标记数 | 用途 |
|---|---|---|
| 0-4、6 | 0 | 不使用 |
| 5 | 1 | 日记本位置 |

## 8. 初始怪物抓取顺序（D17）

游戏设置第 8 步"每名玩家抓一张怪物卡"按**座次顺序依次执行**：

- 顺序：先手玩家 → 顺时针下家 → ...
- 原因：部分怪物技能发动时机为"被玩家抓取时"，同时抓取可能触发未知 bug
- 任务 11 例外：任务开始时**不抓取怪物卡**（起始异常）

## 9. 怪物标记上限（D16）

- 全局怪物标记总数 ≥ 上限 → 玩家失败
- 上限由 `TaskCard.monster_token_limit` 字段决定（默认 30）
- 可由任务/难度配置覆盖
- 上限不写死，改为对局初始化时读取任务卡配置
