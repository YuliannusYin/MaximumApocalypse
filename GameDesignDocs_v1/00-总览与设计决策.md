# 00 - 总览与设计决策

> 本文件是《末日启示录》桌游 PC 重制项目的设计总纲，汇总所有已确认决策与待解决问题。
> 原版文档位于 `桌游实体/` 文件夹，不做任何修改。所有新设计文档位于 `GameDesignDocs/`。

---

## 1. 项目背景与目标

### 1.1 项目背景
将实体桌游《末日启示录》(Maximum Apocalypse) 重制为 Godot 4.7 PC 游戏。

原版是一款 1-6 人合作类后启示录求生桌游，玩家扮演求生者在僵尸/突变体/机器人/外星人四类末日威胁中探索地图、收集物资、击杀怪物，最终完成任务目标并驾驶面包车逃离。

### 1.2 重制必要性
原版为线下真人实体桌游，直接移植到 PC 存在以下问题：
- **口语化/模糊表述**：如"按喜好摆放""便于够到"等无法算法化
- **逻辑相悖**：两份地图文件不一致、术语未统一、饥饿伤害表述矛盾等
- **依赖真人裁判**：如"尽可能平均分配敌人"等需 MC 主观判断
- **物理实体依赖**：骰子、token、卡牌手感等难以直接数字化
- **多人面对面依赖**：交易、协作、同时行动等机制

### 1.3 项目目标
- 优先开发**单人单机模式**，为后续热座/AI 队友/联机模式预留接口
- 保留原版核心玩法：回合制、卡牌驱动、合作求生
- 算法化所有模糊规则，消除原文矛盾
- 程序化生成地图布局，提升重玩性

### 1.4 不修改原版
所有原版文档保留在 `桌游实体/` 文件夹，不做任何修改。新设计文档统一放在 `GameDesignDocs/` 文件夹。

## 2. 已确认的核心架构决策

### D1. 平台模式
**单人单机优先**，为单机热座、AI 队友、多人联机留下接口作为中后期开发计划。

- MVP 阶段：1 人单机，玩家操作 1 名主角
- 接口预留：角色控制系统设计为可扩展（玩家控制 / AI 控制 / 网络控制三种驱动）

### D2. 战斗节奏
**纯回合制**，严格沿用原版。

- 玩家轮流行动，每人 4 行动点
- 怪物在玩家回合结束时自动攻击（无需投骰）
- 顺序：玩家1回合 → 玩家2回合 → ...（单机时仅玩家1）

### D3. 地图布局
**任务卡内置布局 + 地块随机填充**（程序化随机生成接口保留为未来扩展）。

- 任务卡 `map_layout` 字段存储二维数组，定义地图形状（固定排版）
- 数组值含义：`0`=空位 / `1`=任意地块 / `2`=面包车 / `3`=必需地块占位
- 地块内容从 38 个实例池随机抽取填充（保留重玩性）
- 必需地块由任务卡 `required_blocks` 列表按顺序填充到占位值为 `3` 的格子
- 同类型多实例地块采用"类型内随机选实例"策略（如3个城市街道实例随机选1个）
- 程序化随机生成接口保留，未来玩家可选"内置布局"或"随机生成"

### D4. 地块范围
**仅用 `MapBlocks.md`** 的 25 种地块（共 38 个地块实例）。

- 以 `MapBlocks.md` 为唯一地块数据源
- 25 种地块类型：百货商店 / 避难所 / 城市街道 / 电厂 / 河流 / 机场 / 警察局 / 军事基地 / 监狱 / 旷野 / 工厂 / 购物中心 / 加油站 / 绿洲 / 面包车 / 墓地 / 农场 / 强盗营地 / 山 / 隧道 / 森林 / 沙漠 / 游乐园 / 医院 / 坠毁点
- 38 个实例分布：百货1+避难2+城街3+电厂1+河流2+机场1+警局1+军基1+监狱1+旷野2+工厂1+购物1+加油3+绿洲1+面包车1+墓地1+农场2+强盗2+山2+隧道2+森林2+沙漠2+游乐园1+医院1+坠毁1=38

## 3. 已确认的核心内容决策

### D5. 燃料机制
**面包车油箱机制**。

- 面包车有一个油箱，容量由任务卡决定
- 玩家需"移动"到面包车所在地块 + "使用"装备区的"燃料"卡来加油
- 加满指定量后面包车才能发动
- 燃料卡是特殊装备卡，抓取时立即三选一（不消耗行动）：
  - **装备**：进入装备区占用1格
  - **使用**：立即给面包车或弹药类型为"燃料"的武器加油（需在有效地块/目标范围）
  - **弃掉**：`discard()` 进拾荒弃牌堆

### D6. 装备栏容量
**默认 4 格，狗为 0 格**。

- 在求生者类中定义变量 `equipment_slot_capacity: int = 4`
- 狗角色 `equipment_slot_capacity = 0`
- 所有角色统一默认值，无需逐角色配置（除狗外）

### D7. 术语统一
**两个核心函数**：

- `remove()` = 移出本局游戏（不进弃牌堆，不可回收）
  - 涵盖原版"销毁 / 摧毁 / 烧毁"
- `discard()` = 置入弃牌堆（可被效果回收或重洗）
  - 涵盖原版"弃掉 / 弃置 / 弃牌"

### D8. 玩家死亡后装备卡处理
**按装备卡来源区分**：

- **来自求生者牌库的装备卡** → 随求生者死亡一同 `remove()`（移出游戏）
- **来自拾荒牌堆的装备卡** → 跟随其他拾荒卡留在死亡地块（可被其他玩家捡起）

> 实现要点：每张装备卡需记录其来源（`survivor_deck` / `scavenger_deck`），死亡时按来源分别处理。

### D9. 装备卡使用流程
**抓取先进手牌，使用需花行动**。

- 所有装备卡（求生者牌库 + 拾荒牌堆）抓取时先进**求生者手牌**
- 例外：燃料卡等特殊装备卡（见 D5）抓取时立即触发特殊处理，不进手牌
- 玩家需消耗 **1 个行动**"使用"装备牌，才能使其从手牌进入装备区
- 装备栏满时拾荒到装备卡：仍**进手牌**（占手牌位），后续装备时若栏满需先 `discard()` 一个已有装备

### D10. 怪物牌库循环
**怪物弃牌堆可重洗**。

- 怪物卡被消灭或玩家死亡时置入**怪物弃牌堆**
- 当怪物牌库被抓取空时，将怪物弃牌堆重新洗混为新的怪物牌库
- 与拾荒牌库不同（拾荒弃牌堆不可重洗，见 D12）

### D11. 玩家死亡后纠缠怪物处理
**置入怪物弃牌堆**。

- 玩家死亡时，其面前所有纠缠怪物卡置入怪物弃牌堆（不生成怪物标记，不留在地块）
- 怪物弃牌堆在怪物牌库空时重洗（见 D10）
- 携带的拾荒卡留在死亡地块可被捡起（见 D8）

### D12. 拾荒牌库空
**无法拾荒**。

- 某色拾荒牌库抓空后，该色**无法拾荒**
- 地块拾荒标记变灰提示
- 拾荒弃牌堆**不可重洗**（保持原版规则）

### D13. MVP 卡牌库范围
**最小可玩闭环**。

- MVP 阶段仅实现 **1 名角色 + 任务0** 所需的最小卡牌子集
- 验证核心循环（移动/拾荒/战斗/饥饿/燃料/胜利）后逐步扩展
- 后续按任务 1-12 逐步录入卡牌

### D14. 难度变体
**MVP 仅留接口与设计方案**。

- MVP 阶段不实现任何变体（难度变体/单人变体/末日变体/老兵+狗变体）
- 但在架构中预留接口与设计方案，便于后期补充
- MVP 仅实现标准模式

### D15. 老兵+狗
**MVP 不实现**。

- 老兵+狗作为"双角色单位"留待后期实现
- MVP 阶段可选角色池不含老兵
- 后期实现时定义为：选择老兵即同时获得老兵+狗两个角色，共享回合但分别结算生命/饥饿

### D16. 怪物标记上限
**由初始化程序返回值决定**。

- 保持"用完即失败"的游戏规则
- 上限不写死 30，改为对局初始化时调用"怪物标记初始化"程序获取返回值作为上限
- 默认返回 30，可由任务/难度配置覆盖

### D17. 初始怪物抓取顺序
**按座次顺序依次抓取**。

- 游戏设置第8步"每名玩家抓一张怪物卡"按座次顺序依次执行
- 原因：部分怪物技能发动时机为"被玩家抓取时"，同时抓取可能触发未知 bug
- 顺序：先手玩家 → 顺时针下家 → ...

### D18. 任务3/8/9/11 燃料需求
**均无需燃料**。

- 这4个任务胜利条件不涉及回面包车，无需启动面包车
- 任务4 已由 R4 确认（3燃料是交付物）

### D19. 任务2 首领机制
**首领卡放牌库底10张中 + 警察局触发额外效果**。

- 修正原版矛盾：首领卡洗入怪物牌库底 10 张中
- 到达警察局时触发额外效果（不再"到警察局才出首领"）

### D20. 任务5 回合限制
**保持原版 + 倒计时 UI**。

- 拆弹后 3 回合内回面包车
- PC 版加回合数倒计时 UI 提示

### D21. 角色文件 H1 标题
**PC 版数据中修正**。

- 原版 `veteran.md` / `hunter.md` / `cowboy.md` 三个文件 H1 都写成"枪手"
- PC 版数据定义中修正 H1 为正确角色名（老兵 / 猎手 / 枪手）
- 不修改原版文件

### D22. 装备栏大小字段
**统一为 `size: int`**。

- 字段值：0 / 1 / 2
- 取代原版"格装备栏"/"一格装备栏"/"1格装备栏"/"两格装备栏"等不统一格式

### D23. 怪物射程"无"的定义
**只攻击纠缠的那一个玩家**。

- 射程为"无"的怪物只能攻击自己所纠缠的那一个玩家
- 即使该玩家所在格子内还有其他玩家，其他玩家也不受攻击
- 区别于"短距离"（攻击纠缠玩家所在格的所有玩家）

### D24. Skill 通用技能系统
**面向对象的统一技能类**。

- 所有"触发时机 + 条件 + 效果"形态的能力统一抽象为 `Skill` 类（建议继承 Godot `Resource`）
- 应用范围：**角色天赋、装备技能、怪物卡技能、地图块技能**全部使用同一 `Skill` 类
- 各类对象（角色 / 装备卡 / 怪物卡 / 地图块）使用 `skills: list[Skill]` 字段存储技能列表
- 引用方式：**对象引用**（非字符串 ID 查表），Godot 中用 Resource 引用最自然
- 角色不再单独定义 `talent` 字段，天赋作为 `skills` 列表的元素（通常为首元素）
- 技能效果用 `effect_id` 引用 `EffectHandler` 函数表，使 `Skill` 数据可纯数据化（便于 `.tres` 文件定义和序列化）
- `SkillExecutor` 中央执行器负责按 `skill_type` 分发执行（架构思路见 [02-角色系统设计.md](02-角色系统设计.md) 第 11 节）

> 设计动机：原版装备卡技能形态高度一致（行动 / 被动 / 触发时机 / 弃置条件），统一为 `Skill` 类符合 DRY 原则，便于 UI 渲染、序列化、技能执行器共用一套逻辑。

### D25. 重调机制
**仅游戏初始抓牌后一次性使用**。

- 适用时机：仅在游戏开始时求生者抓取初始 4 张手牌后可用，**全游戏过程仅 1 次**
- 操作：玩家可选择将最多 4 张刚抓的初始手牌放到一边，从牌库抓等量新牌，再把放下的牌洗回求生者牌库
- 不消耗行动点
- 后续游戏过程中不再触发重调（与原版"每名玩家可进行一次"理解一致：理解为初始一次性使用）
- 解决 [10-原版问题清单.md](10-原版问题清单.md) 第 10 章新发现问题

## 4. 已确认的核心规则裁决

### R1. 饥饿伤害扣血时机
**按示例逻辑**（原版示例说明的正确逻辑）：

- 饥饿等级 1→5：无副作用
- 饥饿等级达到 6：角色卡翻面，进入饥饿状态，**禁用天赋**（本回合不扣血）
- 饥饿状态下每次再 +1 饥饿时，按序列扣血：**2 → 4 → 6 → 8 → 死亡**
- 饥饿伤害不可减少或避免
- 饥饿值降到 6 以下时翻回正面，恢复天赋
- 重新进入饥饿状态时，饥饿伤害序列重置（从 2 重新开始）

### R2. 回合结束效果触发顺序
**地块 → 饥饿 → 怪物**

1. 先结算地块结束效果（若玩家在此地块且地块有"结束"关键词）
2. 再饥饿 +1（触发饥饿伤害检查）
3. 最后怪物攻击（面前所有怪物造成伤害）

### R3. 怪物"纠缠"机制
**抽到即纠缠**：

- 怪物卡被抽到某玩家面前时，即"纠缠"该玩家
- 纠缠关系持续到：怪物被消灭 / 被卡牌效果赶走 / 玩家死亡
- 怪物不主动移动（除非特殊天赋，如僵尸潜行者）
- 玩家可攻击其他玩家面前的怪物（需在射程内）

### R4. 任务4"核冬天"燃料用途
**3 燃料是任务交付物**。

- 任务4 胜利条件：收集 3 燃料 + 2 脏毯子 + 2 老报纸 + 5 食物，带回**避难所**摧毁所有怪物
- 3 燃料不用于启动面包车，是任务交付物
- 任务4 不需要启动面包车（无燃料字段）

## 5. PC版与原版的核心差异

| 维度 | 原版 | PC版 | 变更原因 |
|---|---|---|---|
| 平台 | 线下实体 | 单人单机 | 用户决策 D1 |

| 销毁/摧毁/烧毁 | 未统一 | `remove()` 统一 | 消除歧义 |
| 弃掉/弃置/弃牌 | 未统一 | `discard()` 统一 | 消除歧义 |
| 死亡装备处理 | 未定义 | 按来源区分 | 补全规则 |
| 饥饿伤害时机 | 表述矛盾 | 按示例逻辑 | 消除矛盾 |
| 回合结束顺序 | 未定义 | 地块→饥饿→怪物 | 补全规则 |
| 怪物纠缠 | 未定义 | 抽到即纠缠 | 补全规则 |
| 装备卡使用流程 | 未明确 | 抓取先进手牌，使用花1行动 | 补全规则 |
| 怪物牌库循环 | 未定义 | 弃牌堆可重洗 | 补全规则 |
| 死亡怪物处理 | 换为怪物标记 | 进怪物弃牌堆 | 重新设计 |
| 拾荒牌库空 | 不重洗弃牌堆 | 无法拾荒 | 补全规则 |
| 技能系统 | 各对象分散定义 | 统一 Skill 类 + skills 列表 | D24 架构 |
