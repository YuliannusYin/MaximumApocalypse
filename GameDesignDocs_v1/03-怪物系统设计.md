# 03 - 怪物系统设计

> 本文件定义 PC 版怪物系统的 Monster 数据结构、4 怪物包总览、僵尸包详细设计（MVP）、纠缠机制、怪物牌库循环。
> 基于原版怪物包文档（`桌游实体/基础版/怪物包/`），应用所有设计决策（见 [00-总览与设计决策.md](00-总览与设计决策.md)）。
> 本文档为设计层面，不含代码。怪物技能通过 SkillExecutor 调度（见 [02-角色系统设计.md](02-角色系统设计.md) 第 11 节）。骰子机制与战斗结算见 [06-战斗系统设计.md](06-战斗系统设计.md)。

## 1. 概述

PC 版怪物系统包含 4 个怪物包共 127 张怪物卡：

| 怪物包 | 总数 | 首领数 | 普通+精英 | 应用任务 | MVP 状态 |
|---|---|---|---|---|---|
| 僵尸包（zombie） | 38 | 2 | 36 | 任务 0、1、2、3 | ✅ 详细设计 |
| 突变体包（mutant） | 28 | 2 | 26 | 任务 4、5、6 | 待后续设计 |
| 外星人包（alien） | 27 | 2 | 25 | 任务 7、8、9 | 待后续设计 |
| 机器人包（robot） | 34 | 2 | 32 | 任务 10、11、12 | 待后续设计 |

核心特征：
- 怪物卡统一使用 `Skill` 类描述天赋（见 D24），无需扩展字段
- 怪物"纠缠"玩家机制（见 R3）
- 怪物射程沿用 02 文档 1.5 节 `Range` 枚举（见 D23）
- 怪物牌库循环：弃牌堆可重洗（见 D10）
- 首领卡不进入普通怪物牌库，由任务卡 `boss_config` 单独配置入场（见 [08-任务系统设计.md](08-任务系统设计.md) 6.2 节）

## 2. Monster 数据结构

### 2.1 字段定义

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `monster_id` | string | 怪物唯一标识 | `"zombie_queen"` |
| `monster_name` | string | 怪物显示名（中文） | `"僵尸女王"` |
| `level` | MonsterLevel | 怪物级别枚举（见 2.2） | `BOSS` |
| `max_hp` | int | 初始血量 | 18 |
| `current_hp` | int | 当前血量（运行时，初始=max_hp） | 18 |
| `damage` | int | 攻击伤害（回合结束对玩家造成固定伤害） | 5 |
| `range` | Range | 射程枚举（见 02 文档 1.5 节，D23） | `NONE` |
| `skills` | list[Skill] | 天赋技能列表（空=无天赋，见 D24） | 见第 6 章 |
| `engaged_player` | Player or null | 纠缠的玩家引用（运行时，见 R3） | - |
| `monster_pack` | MonsterPack | 所属怪物包（见 08 文档 2.4 节） | `ZOMBIE` |

### 2.2 MonsterLevel 枚举

```text
MonsterLevel = BOSS | ELITE | NORMAL
```

| 枚举 | 中文 | 说明 |
|---|---|---|
| `BOSS` | 首领 | 首领卡，由任务卡 `boss_config` 配置入场（见 08 文档 6.2 节） |
| `ELITE` | 精英 | 精英怪，通常有天赋 |
| `NORMAL` | 普通 | 普通怪，通常无天赋 |

> 首领卡不进入普通怪物牌库，由任务卡 `boss_config` 单独配置处理。精英和普通怪洗入怪物牌库。

## 3. 怪物纠缠机制（R3）

### 3.1 纠缠关系

- 怪物卡被抽到某玩家面前时，即"纠缠"该玩家
- 纠缠关系记录在 `monster.engaged_player` 字段
- 纠缠关系持续到：
  - 怪物被消灭（血量归零）
  - 被卡牌效果赶走（如伪装"换掉面前非首领怪物"）
  - 玩家死亡（见 D11，纠缠怪物置入怪物弃牌堆）

### 3.2 纠缠关系与移动

- 纠缠关系绑定**玩家**，不绑定地块
- 玩家移动时，纠缠的怪物卡**跟随玩家**（纠缠关系不解除）
- 怪物**不主动改变**纠缠对象（除非特殊天赋，如僵尸潜行者"移动到血量最低玩家面前"）
- 回合结束怪物攻击时，攻击纠缠的玩家（不管玩家在哪个地块）

### 3.3 怪物卡与地块的关系

- 怪物卡**不占地块格子**（怪物卡在玩家面前，是逻辑概念）
- 地块上的"怪物标记"是独立概念，与怪物卡无关（见第 8 章）
- 玩家可攻击其他玩家面前的怪物（需在射程内，见 06 文档）

## 4. 怪物射程与攻击范围（D23）

怪物射程沿用 02 文档 1.5 节 `Range` 枚举。怪物攻击范围以**纠缠玩家所在格**为中心：

| 射程 | 攻击范围 | 说明 |
|---|---|---|
| `NONE` | 只攻击纠缠的那一个玩家 | 即使同格有其他玩家也不受攻击（D23） |
| `SHORT` | 纠缠玩家所在格的所有玩家 | 同格其他玩家也受攻击 |
| `MEDIUM` | 纠缠玩家所在格+4邻格的所有玩家 | 范围攻击 |
| `LONG` | 纠缠玩家为中心 5×5 减四角范围的所有玩家 | 范围攻击 |

> 怪物攻击在"纠缠玩家的回合结束阶段"触发（见 01 文档回合流程）。射程 MEDIUM/LONG 的怪物会波及范围内的其他玩家（即使不是他们的回合）。

## 5. 怪物牌库循环（D10）

### 5.1 牌库构成

- 每个任务使用一个怪物包（见 08 文档 `monster_pack` 字段）
- 牌库由该怪物包的所有精英+普通怪组成（首领卡不进牌库，由 `boss_config` 单独配置）
- 任务开始时洗混形成怪物牌库

### 5.2 牌库循环

- 怪物卡被消灭或玩家死亡时置入怪物弃牌堆（见 D11）
- 怪物牌库被抓空时，将怪物弃牌堆重新洗混为新的怪物牌库（可重洗，见 D10）
- 与拾荒牌库不同（拾荒弃牌堆不可重洗，见 D12）

### 5.3 抓怪物卡的时机

| 时机 | 来源 | 说明 |
|---|---|---|
| 游戏设置第8步 | 系统初始化 | 每名玩家按座次顺序抓1张怪物卡（见 D17） |
| 地块触发效果 | 地块技能 | 如旷野、墓地、城市街道"抓一张怪物卡" |
| 卡牌效果 | 拾荒卡/装备卡 | 如伏击卡、感应地雷等 |
| 怪物天赋 | 怪物技能 | 如僵尸女王、一大波僵尸等 |
| 任务事件 | 任务系统 | 如首领卡配置触发（见 08 文档 6.2 节） |

## 8. 怪物标记系统

### 8.1 怪物标记 vs 怪物卡

PC 版严格区分两个独立概念：

| 概念 | 来源 | 位置 | 作用 |
|---|---|---|---|
| 怪物卡 | 怪物牌库抓取 | 纠缠玩家面前 | 回合结束攻击玩家；玩家可主动攻击消灭 |
| 怪物标记 | 怪物出生阶段投骰 | 地块上 | 影响潜行检定（每标记-1潜行）；检定失败时移除并抓怪物卡 |

> 怪物卡和怪物标记是两个独立系统。怪物卡不占地块，怪物标记不是卡。

### 8.2 怪物标记生成

回合开始时执行怪物出生阶段（回合流程第1阶段，见 01 文档）：
- 投 2 颗 D6 骰子，点和为 X
- 在 `monster_spawn_value == X` 的地块上放置 1 个怪物标记
- 若多块地块的 `monster_spawn_value` 都等于 X，则全部放置
- 检查怪物标记总数是否达上限（D16），达到则游戏失败

> 骰子机制与投骰细节详见 [06-战斗系统设计.md](06-战斗系统设计.md)。地块的 `monster_spawn_value` 字段见 [04-地图系统设计.md](04-地图系统设计.md) 第 2.1 节。

### 8.3 怪物标记上限（D16）

- 全局怪物标记总数 ≥ `TaskCard.monster_token_limit`（默认 30）→ 玩家失败
- 上限由任务卡配置，可由难度覆盖

### 8.4 怪物标记移除途径

| 途径 | 说明 |
|---|---|
| 潜行检定失败 | 移除该地块所有怪物标记，每移除1个抓1张怪物卡（见 06 文档） |
| 卡牌效果 | 如大炸药、无人机攻击等（见 05 文档） |
| 任务特殊规则 | 如任务 11 清除目标标记地块的怪物标记 |

## 9. 怪物攻击机制

### 9.1 攻击时机

回合结束阶段（回合流程第5阶段，见 01 文档），按 R2 顺序结算：
1. 地块结束效果
2. 饥饿 +1（触发饥饿伤害检查）
3. 怪物攻击：每张纠缠当前回合玩家的怪物卡发起攻击

### 9.2 攻击伤害

- 怪物攻击伤害**固定**（`monster.damage` 字段），不投骰
- 攻击范围由 `monster.range` 决定（见第 4 章）
- 伤害可被装备效果减免（如焊接头盔-1、防弹背心-2）
- 不可减免的伤害：饥饿伤害（见 R1）

> 战斗结算流程（含伤害减免、装备耐久、死亡判定）详见 [06-战斗系统设计.md](06-战斗系统设计.md)。

## 10. 怪物技能与 Skill 类映射

### 10.1 字段映射

怪物天赋 100% 可用 02 文档 1.2 节定义的 `Skill` 类描述，**无需扩展字段**：

| 原版天赋字段 | Skill 类字段 | 说明 |
|---|---|---|
| 触发时机 | `trigger_timing` | 使用 02 文档 1.4 节枚举（ON_MONSTER_DRAWN / ON_KILL / ON_DAMAGE_DEALT 等） |
| 触发效果 | `effect` + `effect_id` | effect 供 UI 显示，effect_id 供 SkillExecutor 调用 |
| - | `skill_type` | 怪物天赋均为 `TRIGGER`（自动触发，不消耗行动） |
| - | `condition` | 自然语言描述触发条件 |
| - | `action_cost` | 怪物天赋均为 0 |
| - | `discard_self` | 怪物天赋均为 false（天赋不弃置怪物卡本身） |

### 10.2 EffectHandler 函数表（僵尸包示例）

```text
EffectHandler = {
    # 僵尸女王
    "killer_draw_monster_1":              draw_monster_card(target="killer"),
    # 一大波僵尸
    "spawn_tokens_adjacent_3_and_draw_1": spawn_tokens_to_adjacent(count=3, draw_card=true),
    # 僵尸步行者（精英）
    "drawer_draw_monster_1":              draw_monster_card(target="drawer"),
    # 僵尸潜行者（精英）
    "move_to_lowest_hp_player":           change_engaged_player(target="lowest_hp"),
}
```

> effect_id 命名规范见 02 文档 11.4 节。

### 10.3 不扩展 Skill 字段的理由

曾考虑增加 `target_scope` 和 `parameters` 字段，但 MVP 阶段不采用，原因：
- `condition` 字段已用自然语言描述触发条件，足够覆盖僵尸包所有天赋
- `effect_id` 指向 EffectHandler 中的组合函数，复杂逻辑（如"放标记+抓卡"）可在函数内实现
- 遵循"避免过度设计"原则，后续怪物包出现无法描述的情况再考虑扩展
