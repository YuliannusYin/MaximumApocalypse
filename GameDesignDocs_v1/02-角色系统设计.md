> 本文件定义 PC 版角色系统的数据结构、Skill 通用技能类（见 D24）、装备栏、饥饿影响、死亡处理，并列出 MVP 阶段可选的 5 名角色。

## 1. 角色数据结构

### 1.1 角色字段定义

每个角色包含以下字段：

| 字段 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `character_id` | string | 角色唯一标识（文件名） | `"surgeon"` |
| `character_name` | string | 角色中文名 | `"外科医生"` |
| `max_hp` | int | 生命值上限 | 23 |
| `current_hp` | int | 当前生命值（初始=max_hp） | 23 |
| `stealth_normal` | int | 正常状态潜行值 | 8 |
| `stealth_hungry` | int | 饥饿状态潜行值 | 7 |
| `skills` | list[Skill] | 技能列表（天赋通常为首元素；见 D24 与 1.2） | 见 1.2 |
| `equipment_slot_capacity` | int | 装备栏容量（默认4，狗为0，见 D6） | 4 |
| `current_hunger` | int | 当前饥饿等级（初始1） | 1 |
| `is_hungry` | bool | 是否处于饥饿状态（饥饿值≥6） | false |
| `equipment_zone` | list[EquipmentCard] | 装备区（已装备卡列表） | [] |
| `hand` | list[Card] | 当前手牌（上限10） | [] |
| `deck` | list[SurvivorCard] | 个人求生者牌库 | [] |
| `discard_pile` | list[SurvivorCard] | 个人求生者弃牌堆 | [] |
| `poison_stacks` | int | 中毒层数 | 0 |
| `stunned_monsters` | list[Monster] | 被击晕的怪物列表（临时状态） | [] |
| `current_location` | MapBlock | 当前所在地块 | - |

### 1.2 Skill 类定义（见 D24）

所有"触发时机 + 条件 + 效果"形态的能力统一抽象为 `Skill` 类（建议继承 Godot `Resource`）。
应用范围：**角色天赋、装备技能、怪物卡技能、地图块技能**全部使用同一 `Skill` 类。

```text
Skill {
    skill_id: string               # 技能唯一ID（如 "cowboy_fast_draw"）
    name: string                   # 技能显示名（中文）
    skill_type: SkillType          # 技能类型枚举（见 1.3）
    trigger_timing: TriggerTiming  # 触发时机枚举（见 1.4）
    range: Range                   # 射程枚举（见 1.5）
    condition: string              # 触发条件（自然语言描述，可为空）
    effect: string                 # 效果描述（自然语言，供 UI 显示）
    effect_id: string              # 效果处理函数ID（供 SkillExecutor 调用，见第 11 节）
    action_cost: int               # 消耗行动点（PASSIVE/ON_DISCARD=0，ACTIVE 通常=1）
    discard_self: bool             # 是否需弃置此卡才生效（ON_DISCARD=true，部分 TRIGGER=true）
}
```

字段说明：
- `effect` 与 `effect_id` 分离：前者供 UI 显示，后者供 SkillExecutor 调用 `EffectHandler` 函数表中的处理函数，使 Skill 数据可纯数据化
- `condition` 与 `effect` 在设计阶段均为自然语言描述，代码阶段再细化为可执行条件 / 效果

### 1.3 SkillType 枚举

```text
SkillType = ACTIVE | PASSIVE | TRIGGER | ON_DISCARD
```

| 枚举 | 说明 | 示例 |
|---|---|---|
| `ACTIVE` | 主动技能，玩家消耗行动点使用 | 消防员"拳打"、外科医生"缝合" |
| `PASSIVE` | 被动技能，持续生效或自动应用 | 焊接头盔"受伤-1"、背包"+1装备栏" |
| `TRIGGER` | 触发技能，特定时机自动触发（不消耗行动） | 感应地雷"抓怪物卡时弃此装备造成7伤害"、游侠帽"装备时回复3生命" |
| `ON_DISCARD` | 弃置技能，玩家主动弃置此卡以触发效果（不消耗行动） | 梯子"弃置跳过抓怪物卡"、伪装"弃置受3伤换掉面前怪物" |

> TRIGGER 与 ON_DISCARD 的区别：
> - `TRIGGER`：触发时机一到**自动触发**，宿主卡牌不一定弃置（除非 `discard_self=true`）
> - `ON_DISCARD`：玩家**主动选择**弃置此卡才触发效果

### 1.4 TriggerTiming 枚举（可扩展）

```text
TriggerTiming =
    PASSIVE |              # 持续生效（与 SkillType.PASSIVE 配合）
    ACTIVE_USE |           # 玩家主动使用（与 SkillType.ACTIVE 配合）
    ON_GAME_START |        # 游戏开始时
    ON_TURN_START |        # 回合开始时
    ON_TURN_END |          # 回合结束时
    ON_DAMAGE_DEALT |      # 造成伤害时
    ON_DAMAGE_TAKEN |      # 受到伤害时
    ON_HUNGER_DAMAGE |     # 受饥饿伤害时
    ON_CARD_DRAWN |        # 抓牌时
    ON_MONSTER_DRAWN |     # 抓怪物卡时
    ON_EQUIP |             # 装备进入装备区时
    ON_KILL |              # 杀死怪物时
    ON_DISCARD_CHOICE      # 玩家主动弃置以触发（与 SkillType.ON_DISCARD 配合）
```

> 此枚举随技能录入逐步扩展，MVP 阶段先支持上述触发时机。`ON_DISCARD_CHOICE` 不算严格意义的"事件触发"，而是 SkillExecutor 在玩家选择弃置装备时检查 `SkillType.ON_DISCARD` 技能的入口。

### 1.5 射程枚举（Range）

```text
Range = NONE | SHORT | MEDIUM | LONG
```

- `NONE`：无射程（玩家无法主动指定；怪物只攻击纠缠玩家，见 D23）
- `SHORT`：短距离（当前格）
- `MEDIUM`：中距离（当前格+相邻4格）
- `LONG`：长距离（距离2十字+对角线1）

## 2. 技能系统（角色部分）

### 2.1 角色技能规则
- 每个角色的 `skills` 列表首元素为角色天赋（其余元素预留后续扩展，如升级天赋）
- 天赋在**非饥饿状态**可用
- 饥饿状态（饥饿值≥6）时**禁用天赋**（见 R1）
- 主动天赋通常消耗 1 行动点（被动/触发天赋不消耗）
- 天赋效果由 SkillExecutor 调度执行（见第 11 节）

### 2.2 角色天赋触发时机分类

| 触发时机 | 说明 | 示例 |
|---|---|---|
| 主动 | 玩家主动消耗行动点使用 | 消防员"拳打" |
| 被动 | 持续生效，无需消耗 | （无 MVP 角色） |
| 游戏开始 | 游戏初始化时触发 | 枪手"快速拔枪"第一段 |
| 受饥饿伤害 | 受饥饿伤害时触发 | 枪手"快速拔枪"第二段 |

## 3. 装备栏系统

### 3.1 装备栏容量
- 默认容量：4 格（见 D6）
- 狗角色：0 格
- 容量在角色数据中定义，不可动态改变（部分装备可扩展容量，如"背包"被动+1格）

### 3.2 装备卡大小（见 D22）
统一为 `size: int` 字段：
- `0`：不占装备位（如柯尔特手枪）
- `1`：占 1 格
- `2`：占 2 格

### 3.3 装备卡使用流程（见 D9）
1. 装备卡抓取时先进**手牌**
2. 玩家消耗 1 行动点"使用"装备卡 → 进入装备区
3. 例外：燃料卡等特殊装备（见 D5）抓取时立即三选一，不进手牌
4. 装备栏满时装备：需先 `discard()` 一个已有装备

### 3.4 装备区来源标记（见 D8）
每张装备卡记录来源：
- `survivor_deck`：来自求生者牌库
- `scavenger_deck`：来自拾荒牌堆

死亡时按来源分别处理（见第 6 章）。

## 4. 饥饿状态对角色的影响

### 4.1 饥饿状态切换
- 饥饿值 1-5：正常状态，天赋可用，使用 `stealth_normal`
- 饥饿值 ≥6：饥饿状态，天赋禁用，使用 `stealth_hungry`

### 4.2 状态切换效果
- 进入饥饿状态（饥饿值达到6）：
  - 角色卡翻面（UI 表现：边框变红/图标变色）
  - 天赋禁用
  - 潜行值降为 `stealth_hungry`
  - 本回合不扣血（见 R1）
- 退出饥饿状态（饥饿值降到5及以下）：
  - 角色卡翻回正面
  - 天赋恢复
  - 潜行值恢复 `stealth_normal`
  - 饥饿伤害序列重置（下次进入饥饿从 2 重新开始）

## 5. 角色死亡处理

### 5.1 死亡触发
- 生命值归零
- 求生者牌库空，无法抓牌

### 5.2 死亡处理流程
1. **装备卡处理**（见 D8）：
   - `survivor_deck` 来源的装备卡 → `remove()`（移出游戏）
   - `scavenger_deck` 来源的装备卡 → 留在死亡地块（可被捡起）
2. **纠缠怪物处理**（见 D11）：面前所有纠缠怪物置入怪物弃牌堆
3. **携带拾荒卡**：留在死亡地块，可被同地块玩家花 1 行动捡起
4. **个人求生者牌库与弃牌堆**：随玩家死亡移出游戏

### 5.3 掉落卡捡起
- 同地块玩家花 1 行动可捡起 1 张掉落卡
- 掉落卡包括：`scavenger_deck` 来源的装备卡、拾荒卡

## 7. 角色卡牌库结构

### 7.1 牌库构建
- 每个角色有独立的求生者牌库
- 牌库由该角色的所有卡牌组成（见各角色牌库清单）
- 游戏开始时洗混
- 玩家从牌库抓牌到手牌

### 7.2 牌库循环
- 牌库空时玩家被淘汰（任意需抓牌时）
- 求生者弃牌堆在牌库空时可重洗（具体规则见各卡牌效果，如机械师"神通广大"涉及弃牌堆）

### 7.3 牌库与装备来源（见 D8）
- 牌库中的装备卡 → 来源标记为 `survivor_deck`
- 拾荒牌堆中的装备卡 → 来源标记为 `scavenger_deck`
- 死亡时按来源分别处理

## 8. 角色特殊机制说明

### 8.1 枪手"快速拔枪"特殊处理
- 由 `cowboy_fast_draw_init`（ON_GAME_START）和 `cowboy_fast_draw_recall`（ON_HUNGER_DAMAGE）两个 Skill 实现
- 游戏开始时 SkillExecutor 自动触发 `equip_colt_from_deck`：将柯尔特手枪从牌库直接放入装备区（不进手牌、不消耗行动、size=0 不占装备位）
- 受饥饿伤害时 SkillExecutor 自动触发 `recall_colt_to_deck`：将装备区或弃牌堆中的柯尔特手枪移回牌库并洗混
- 实现要点：EffectHandler 中的 `equip_colt_from_deck` 函数需处理"柯尔特手枪不在牌库"的兜底情况（如已在装备区/弃牌堆）

### 8.2 消防员"野地夹克"特殊机制
- 装备时若饥饿值 <6，可主动 +1 饥饿恢复 1 行动数
- 此行为不消耗每回合的行动点数（额外行动数）
- 实现要点：需追踪"野地夹克恢复的行动数"与"标准行动点"分开

### 8.3 装备扩展容量
- 部分装备可扩展装备栏容量（如猎手"背包"被动 +1 格）
- 实现要点：装备栏容量 = `equipment_slot_capacity` + 装备区中所有 +容量 效果之和

## 11. SkillExecutor 架构思路（见 D24）

本节定义 Skill 系统的中央执行器架构。所有 Skill 的实际效果调度由 SkillExecutor 统一负责，Skill 数据本身不含逻辑代码（遵循数据/逻辑分离原则）。

### 11.1 组件构成

```text
SkillExecutor 架构 {
    SkillRegistry      # 技能注册表：按 skill_id 索引所有 Skill 实例
    SkillExecutor      # 中央执行器：按 skill_type 分发，按 trigger_timing 匹配
    EffectHandler      # 效果处理函数表：effect_id → 处理函数
    ConditionChecker   # 条件检查器：解析 condition 字段并判定是否触发
}
```

| 组件 | 职责 | 实现建议 |
|---|---|---|
| `SkillRegistry` | 存储所有 Skill 实例（按 `skill_id` 索引）；支持运行时查询 | Godot autoload 单例，启动时从 `data/skills/` 加载所有 `.tres` |
| `SkillExecutor` | 接收游戏事件，扫描相关对象的 `skills` 列表，分发到 `EffectHandler` | Godot autoload 单例，监听全局事件总线 |
| `EffectHandler` | 集中实现所有具体效果函数（按 `effect_id` 查表调用） | 单一 GDScript 文件，函数名 = `effect_id` |
| `ConditionChecker` | 解析 `condition` 字段，判定是否满足触发条件 | MVP 阶段可用 `if-elif` 链，后期可改为脚本表达式 |

### 11.2 执行流程

```text
1. 游戏事件发生（如 ON_TURN_START）
   ↓
2. SkillExecutor 扫描所有相关对象的 skills 列表
   - 对象来源：当前回合玩家 + 其装备区 + 其所在地块 + 地块上的怪物
   ↓
3. 对每个匹配 trigger_timing 的 Skill：
   ↓
4. 调用 ConditionChecker 检查 condition 是否满足
   - 不满足 → 跳过
   - 满足 → 进入下一步
   ↓
5. 调用 EffectHandler[effect_id] 执行效果
   - ACTIVE 技能：玩家主动点击触发，先扣 action_cost 再执行
   - PASSIVE/TRIGGER 技能：事件触发时自动执行
   - ON_DISCARD 技能：玩家主动弃置宿主卡时触发
   ↓
6. 根据 discard_self 决定是否弃置宿主卡
   - true → 调用 discard() 将宿主卡置入对应弃牌堆
   - false → 宿主卡保留
```

### 11.3 分发逻辑

| SkillType | 触发方式 | action_cost | discard_self | 示例 |
|---|---|---|---|---|
| `PASSIVE` | 事件自动触发，持续生效 | 0 | false | 焊接头盔"受伤-1" |
| `ACTIVE` | 玩家主动消耗 action_cost 使用 | 通常=1 | false | 消防员"拳打" |
| `TRIGGER` | trigger_timing 事件发生时自动触发 | 0 | 视情况 | 感应地雷（discard_self=true）、游侠帽（false） |
| `ON_DISCARD` | 玩家主动弃置宿主卡时触发 | 0 | true（必然） | 梯子、伪装 |

### 11.4 EffectHandler 函数表（MVP 示例）

```text
EffectHandler = {
    # 角色天赋
    "heal_target_1":              heal_target(amount=1),
    "damage_target_2":            damage_target(amount=2),
    "draw_equip_from_discard":    draw_card_from_pile(pile_type="discard", card_type="equipment"),
    "reveal_adjacent_blocks_2":   reveal_adjacent_blocks(count=2, trigger_effects=false),
    # 枪手快速拔枪
    "equip_colt_from_deck":       move_card_to_zone(card_id="colt_revolver", source="deck", target="equipment_zone"),
    "recall_colt_to_deck":        move_card_to_deck(card_id="colt_revolver", sources=["equipment_zone", "discard_pile"], shuffle=true),
    # 装备技能（示例，详见 05-卡牌系统设计.md）
    "damage_target_7_on_monster_drawn": damage_target(amount=7, trigger="on_monster_drawn", discard_self=true),
    "reduce_damage_taken_1":      modify_damage(delta=-1, scope="all"),
    # ...
}
```

> `effect_id` 命名规范：动词_对象_参数 或 动词_数量，全小写下划线分隔，避免重名。
> `EffectHandler` 函数实现集中在单一 GDScript autoload 中，便于维护与扩展。

### 11.5 数据/逻辑分离原则

- **Skill 数据**（`.tres` 文件）：只含字段定义，无逻辑代码。每个 Skill 一个 `.tres` 文件，存放于 `data/skills/` 目录
- **EffectHandler 逻辑**：所有效果实现集中在 `EffectHandler` 单例，按 `effect_id` 查找对应函数
- **优势**：
  - 新增技能只需新建 `.tres` 文件 + 在 `EffectHandler` 中添加处理函数，无需修改 `Skill` 类
  - UI 渲染只需读取 Skill 数据，与执行逻辑解耦
  - 平衡性调整（修改 `effect_id` 函数实现）不影响数据文件
  - 便于后续任务/扩展包新增技能

### 11.6 MVP 阶段实现优先级

1. **基础架构**：SkillRegistry + SkillExecutor + EffectHandler 骨架
2. **5个角色天赋**（见 6.1-6.5）的 effect_id 实现
3. **关键装备技能**（如感应地雷、焊接头盔、梯子、伪装）的 effect_id 实现
4. **关键地块技能**（见 04-地图系统设计.md）的 effect_id 实现
5. 后续按任务逐步扩展 EffectHandler 函数表
