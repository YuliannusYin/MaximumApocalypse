# 09 - 术语表

> 本文件汇总 PC 版《末日启示录》设计文档中使用的所有核心术语、英文 ID、枚举值、字段名、EffectHandler 函数 ID 与命名规范。
> 用于在开发阶段统一术语、避免歧义、便于检索。
> 本文为索引性质文档，不定义新规则；规则定义见对应系统设计文档。

---

### 1.2 卡牌去向函数（D7）

| 函数 | 含义 | 涵盖原版术语 |
|---|---|---|
| `remove()` | 移出本局游戏（不进弃牌堆，不可回收） | 销毁 / 摧毁 / 烧毁 |
| `discard()` | 置入弃牌堆（可被效果回收或重洗） | 弃掉 / 弃置 / 弃牌 |

> 详细规则见 [05-卡牌系统设计.md](05-卡牌系统设计.md) 7 节。

| 进入 | `ON_ENTER` | 玩家进入地块时（每次） | [04](04-地图系统设计.md) 3 |

## 7. 枚举值汇总

### 7.1 CardType（卡牌类型）

```text
CardType = ACTION | EQUIPMENT | ITEM
```

| 枚举 | 说明 | 打出后去向 |
|---|---|---|
| `ACTION` | 行动卡，打出消耗 1 行动，结算后弃掉 | `discard()` 进拾荒弃牌堆 |
| `EQUIPMENT` | 装备卡，进手牌，使用花 1 行动进入装备区 | 装备区 / 手牌 |
| `ITEM` | 任务物品卡，占手牌位，用于任务结算 | 手牌（直到交付） |

```text
ScavengerColor = RED | BLUE | GREEN | GRAY
```

### 7.7 SkillType（技能类型）

```text
SkillType = ACTIVE | PASSIVE | TRIGGER | ON_DISCARD
```

| 枚举 | 说明 | action_cost | 示例 |
|---|---|---|---|
| `ACTIVE` | 主动技能，玩家消耗行动点使用 | 通常=1 | 消防员"拳打"、外科医生"缝合" |
| `PASSIVE` | 被动技能，持续生效或自动应用 | =0 | 焊接头盔"受伤-1"、背包"+1装备栏" |
| `TRIGGER` | 触发技能，特定时机自动触发 | =0 | 感应地雷、游侠帽 |
| `ON_DISCARD` | 弃置技能，玩家主动弃置此卡以触发效果 | =0 | 梯子、伪装 |

> TRIGGER 自动触发，宿主卡牌不一定弃置（除非 `discard_self=true`）；ON_DISCARD 玩家主动选择弃置才触发。

定义位置：[02-角色系统设计.md](02-角色系统设计.md) 1.3

### 7.8 TriggerTiming（触发时机）

完整枚举 = 02 文档 1.4 节 13 项 + 04 文档 3 节 5 项增补 = **18 项**。

```text
TriggerTiming =
    # 通用时机（02 文档 1.4 节）
    PASSIVE |              # 持续生效
    ACTIVE_USE |           # 玩家主动使用
    ON_GAME_START |        # 游戏开始时
    ON_TURN_START |        # 回合开始时
    ON_TURN_END |          # 回合结束时
    ON_DAMAGE_DEALT |      # 造成伤害时
    ON_DAMAGE_TAKEN |      # 受到伤害时
    ON_HUNGER_DAMAGE |     # 受饥饿伤害时
    ON_CARD_DRAWN |        # 抓牌时
    ON_MONSTER_DRAWN |     # 抓怪物卡时
    ON_EQUIP |             # 装备进入装备区时
    ON_KILL |              # 杀死怪物时
    ON_DISCARD_CHOICE |    # 玩家主动弃置以触发（配合 SkillType.ON_DISCARD）
    # 地块专用时机（04 文档 3 节增补）

    ON_LEAVE |             # 玩家离开地块时
    ON_ACTION |            # 玩家在地块上消耗行动执行地块行动

```

定义位置：[02-角色系统设计.md](02-角色系统设计.md) 1.4 + [04-地图系统设计.md](04-地图系统设计.md) 3

### 10.1 ID 命名规范

| 类型 | 格式 | 示例 |
|---|---|---|
| `character_id` | 全小写英文，与原版文件名一致 | `surgeon` / `cowboy` |
| `monster_id` | 全小写英文+下划线，含怪物包前缀 | `zombie_queen` / `zombie_walker_elite` |

| `skill_id` | 全小写英文+下划线，含角色/装备前缀 | `cowboy_fast_draw` / `surgeon_suture` |
| `effect_id` | 动词_对象_参数 或 动词_数量，全小写下划线分隔，避免重名 | `heal_target_1` / `reduce_hunger_self_2` |
| `instance_id` | `block_id` + `_` + 序号 | `general_store_1` / `gas_station_3` |
| `mission_id` | 全小写英文+下划线 | （详见 08 文档 3 章） |

### 10.2 字段命名规范

- 所有字段使用 `snake_case`（小写下划线）
- 运行时字段（实例字段）与静态字段（数据字段）分离
- 布尔字段以 `is_` / `has_` 开头
- 当前值字段以 `current_` 开头（与 `max_` 配对）

### 10.3 枚举命名规范

- 所有枚举值使用 `UPPER_SNAKE_CASE`（大写下划线）
- 枚举名使用 `PascalCase`（如 `SkillType` / `TriggerTiming`）
- `NONE` / `NULL` 仅用于"无"或"空"含义，不用于错误状态

### 10.4 文档交叉引用规范

- 简写格式：`[文档编号-文档名](文档编号-文档名.md) 章节`

## 11. 数据/逻辑分离原则（D24 落地）

### 11.1 文件结构

| 目录 | 内容 |
|---|---|
| `data/skills/` | Skill 数据文件（每个 Skill 一个 `.tres` 文件） |
| `data/characters/` | 角色数据文件（每个角色一个 `.tres` 或目录） |
| `data/monsters/` | 怪物数据文件（按怪物包分子目录） |
| `data/blocks/` | 地块数据文件 |
| `data/cards/` | 卡牌数据文件（按 ScavengerColor 分子目录） |
| `data/missions/` | 任务数据文件 |
| `autoload/EffectHandler.gd` | 所有 effect_id 处理函数集中实现 |

### 11.2 SkillRegistry + SkillExecutor + EffectHandler 三层架构

- **SkillRegistry**：管理所有 Skill 数据的注册表
- **SkillExecutor**：中央执行器，按 `skill_type` 分发执行
- **EffectHandler**：所有效果实现的集中地，按 `effect_id` 查找对应函数

定义位置：[02-角色系统设计.md](02-角色系统设计.md) 11

### 11.3 数据/逻辑分离优势

- 新增技能只需新建 `.tres` 文件 + 在 `EffectHandler` 中添加处理函数，无需修改 `Skill` 类
- UI 渲染只需读取 Skill 数据，与执行逻辑解耦
- 平衡性调整（修改 `effect_id` 函数实现）不影响数据文件
- 便于后续任务/扩展包新增技能
