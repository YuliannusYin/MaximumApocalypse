# Apocrypha 项目规则

## 代码备注规则（强制）

AI Agent 在对本项目的代码进行操作时，必须严格遵守以下备注规则。

### 1. 备注时机

- **写代码后**：每次新增代码文件、新增函数、新增类、新增成员变量时，必须在对应位置添加备注。
- **修改代码后**：每次修改已有代码逻辑、修复 Bug、重构代码时，必须在修改处添加或更新备注。

### 2. 备注格式

采用「简洁标签式」格式，包含 **操作类型标签** + **日期** + **简述**：

```
# [标签] YYYY-MM-DD: 简述内容
```

#### 支持的标签

| 标签     | 适用场景                                   |
| -------- | ------------------------------------------ |
| `[新增]` | 新建文件、新增函数、新增类、新增变量等     |
| `[修改]` | 修改已有逻辑、调整实现方式、重构代码       |
| `[修复]` | 修复 Bug                                   |
| `[优化]` | 性能优化、代码质量提升、可读性改善         |
| `[删除]` | 删除代码、移除功能（在删除处或相邻处说明） |
| `[重构]` | 结构性重构，不改变外部行为                 |

### 3. 备注位置示例（GDScript）

```gdscript
# [新增] 2026-06-23: 玩家移动控制器，基于 CharacterBody3D
extends CharacterBody3D

# [新增] 2026-06-23: 移动速度（单位：米/秒）
@export var move_speed: float = 5.0

# [新增] 2026-06-23: 处理玩家输入并应用移动
func _physics_process(delta: float) -> void:
    var input_dir := Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
    velocity = Vector3(input_dir.x, 0, input_dir.y) * move_speed
    move_and_slide()

# [修复] 2026-06-23: 修正跳跃后重力计算错误
func apply_gravity(delta: float) -> void:
    velocity.y -= 9.8 * delta
```

### 4. 备注要求

1. **简述内容**必须清晰说明「做了什么」，避免无意义的描述（如「更新代码」）。
2. **日期**使用 ISO 8601 格式 `YYYY-MM-DD`。
3. **文件级备注**：每个新建的 `.gd` 文件顶部必须有文件级备注，说明文件职责。
4. **函数级备注**：新增或修改的函数必须有备注，说明函数用途。
5. **行内备注**：仅当逻辑复杂、不直观时才添加行内备注，避免过度备注。
6. **修改已有代码**时，若原代码已有备注，应更新原备注的日期和内容，而非叠加新备注。

### 5. 其他约定

- 本项目使用 **Godot 4.7** + **GDScript**，注释统一使用 `#`。
- 代码文件统一使用 **UTF-8** 编码、**LF** 换行符（见 `.editorconfig`）。
- 遵循 Godot 官方 GDScript 风格指南。
