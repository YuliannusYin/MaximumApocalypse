## 1. 项目概述

### 1.1 项目背景

将实体桌游《末日启示录》(Maximum Apocalypse) 重制为 Godot 4.7 + GDScript PC 游戏。

原版是一款 1-6 人合作类后启示录求生桌游，玩家扮演求生者在僵尸/突变体/机器人/外星人四类末日威胁中探索地图、收集物资、击杀怪物，最终完成任务目标并驾驶面包车逃离。

PC 版核心目标：
- 优先开发**单人单机模式**，为后续热座/AI 队友/联机模式预留接口
- 保留原版核心玩法：回合制、卡牌驱动、合作求生
- 算法化所有模糊规则，消除原文矛盾
- 程序化生成地图布局，提升重玩性

## 3. 沿用 v1 的核心决策（基线 D1-D25 + R1-R4）

### 3.1 架构层（D1-D4）

| # | 决策 | 摘要 |
|---|---|---|
| **D1** | 平台模式 | 单人单机优先，为热座/AI 队友/联机留接口。MVP 阶段 1 人单机操作 1 名主角 |
| **D2** | 战斗节奏 | 纯回合制，玩家轮流行动每人 4 行动点，怪物在玩家回合结束时自动攻击 |
| **D3** | 地图布局 | 任务卡内置布局 + 地块随机填充。`map_layout` 二维数组（0=空位/1=任意地块/2=面包车/3=必需地块占位） |
| **D4** | 地块范围 | 仅用 `MapBlocks.md` 的 25 种地块类型 / 38 个地块实例 |

### 3.2 内容层（D5-D25）

| # | 决策 | 摘要 |
|---|---|---|
| **D5** | 燃料机制 | 面包车油箱机制，燃料卡抓取时立即三选一（装备/使用/弃掉），不消耗行动 |
| **D6** | 装备栏容量 | 默认 4 格，狗为 0 格 |
| **D7** | 术语统一 | `remove()` = 移出本局游戏（不可回收）；`discard()` = 置入弃牌堆（可回收） |
| **D8** | 玩家死亡装备卡处理 | 按来源区分：求生者牌库来源 → remove；拾荒牌堆来源 → 留在死亡地块可被捡起 |
| **D9** | 装备卡使用流程 | 抓取先进手牌，使用需花 1 行动；燃料卡例外（D5 三选一） |
| **D10** | 怪物/求生者牌库循环 | 怪物弃牌堆可重洗；求生者弃牌堆可重洗（牌库空时玩家淘汰） |
| **D11** | 玩家死亡纠缠怪物处理 | 置入怪物弃牌堆（不生成怪物标记，不留在地块） |
| **D12** | 拾荒牌库空 | 该色无法拾荒；拾荒弃牌堆不可重洗 |
| **D13** | MVP 卡牌库范围 | 最小可玩闭环：1 名角色 + 任务 0 所需卡牌子集 |
| **D14** | 难度变体 | MVP 仅留接口与设计方案，不实现任何变体 |
| **D15** | 老兵+狗 | MVP 不实现，后期作为"双角色单位"扩展 |
| **D16** | 怪物标记上限 | 由初始化程序返回值决定，默认 30，可由任务/难度配置覆盖 |
| **D17** | 初始怪物抓取顺序 | 按座次顺序依次抓取（避免同时抓取触发未知 bug） |
| **D18** | 任务3/8/9/11 燃料需求 | 均无需燃料（胜利条件不涉及回面包车） |
| **D19** | 任务2 首领机制 | 首领卡洗入怪物牌库底 10 张中 + 到达警察局时触发额外效果 |
| **D20** | 任务5 回合限制 | 拆弹后 3 回合内回面包车，PC 版加倒计时 UI |
| **D21** | 角色文件 H1 标题 | PC 版数据中修正原版三处 H1 错误（不修改原版文件） |
| **D22** | 装备栏大小字段 | 统一为 `size: int`（0/1/2），取代原版不统一格式 |
| **D23** | 怪物射程"无"的定义 | 只攻击纠缠的那一个玩家（即使同格有其他玩家也不受攻击） |
| **D24** | Skill 通用技能系统 | 所有"触发时机+条件+效果"形态的能力统一抽象为 Skill 类，应用范围：角色天赋/装备技能/怪物卡技能/地块技能 |
| **D25** | 重调机制 | 仅游戏初始抓牌后一次性使用，全游戏过程仅 1 次 |

### 3.3 规则裁决（R1-R4）

| # | 裁决 | 摘要 |
|---|---|---|
| **R1** | 饥饿伤害扣血时机 | 饥饿等级 1-5 无副作用；达到 6 翻面进入饥饿状态禁用天赋（本回合不扣血）；饥饿状态下每次再 +1 按序列扣血 **2→4→6→8→死亡**；饥饿伤害不可减免；重新进入饥饿状态序列重置 |
| **R2** | 回合结束效果触发顺序 | 地块 → 饥饿 → 怪物 |
| **R3** | 怪物"纠缠"机制 | 抽到即纠缠该玩家，持续到怪物被消灭/被赶走/玩家死亡；怪物不主动移动（除非特殊天赋） |
| **R4** | 任务4"核冬天"燃料用途 | 3 燃料是任务交付物，不用于启动面包车 |

## 4. PC 版（v2）新增决策

### 4.1 Skill 系统（D26-D32）

| # | 决策 | 出处文档 | 摘要 |
|---|---|---|---|
| **D26** | 三国杀 ECA 模式 | 01-Skill §1 | trigger（时机）→ filter（条件）→ content（动作）三段式 |
| **D27** | SkillType 三类 | 01-Skill §2 | TRIGGER（触发技）/ ACTIVE（主动技）/ DISCARD（弃牌技）；MA 不需要视为技/距离技/禁止技 |
| **D28** | forced 锁定技标记 | 01-Skill §1/§5 | `forced: bool` 字段标记 TRIGGER 是否强制发动；替代 v1 的 PASSIVE 类型，统一用 TRIGGER + forced=true 表达持续型被动 |
| **D29** | 数据 + 逻辑分离 | 01-Skill §1 | Skill 是抽象 Resource 子类（.gd 逻辑）；卡牌/角色/地块的 .tres 数据里只存 `skill_ids: Array[StringName]`，运行时由 SkillRegistry 查表 |
| **D30** | 规则集中 + UI 信号 | 01-Skill §1 | SkillExecutor 在各时机点扫描已注册 Skill 调用 filter/execute；状态变更通过信号驱动 UI |
| **D31** | owner 显式传入 | 01-Skill §1/§5 | filter/execute 第二参数为技能拥有者（解决"装备技能给本人用"等问题） |
| **D32** | GameEvent 三字段区分 | 01-Skill §4 | `player` = 当前回合玩家；`source` = 动作发出方；`target` = 动作承受方；三者可重合 |

## 6. v1 与 PC 版（v2）核心差异

| 维度 | v1 设计 | PC 版（v2）实际 | 变更原因 |
|---|---|---|---|
| 文档组织 | 按系统分（10 份） | 按基类分（12 份，自底向上） | 用户要求从底层类开始 |
| MapBlock 继承 | 继承 Entity | 单独继承 Node | MapBlock 无生命值概念，避免强制实现 take_damage/die |
| 被动技/触发技 | PASSIVE / TRIGGER 二分 | 统一用 TRIGGER + forced=true | 与三国杀锁定技一致，简化类型 |
| SkillExecutor 章节 | 在 02-角色系统设计.md 第 11 节 | 在 01-Skill.md 第 8 节（待 13-SkillExecutor.md 单独成文） | Skill 是更底层的基类 |
| 牌堆基类 | 抽象 Deck/DiscardPile 基类 | 不抽象，独立类 + DeckUtils 静态工具 | ponytail YAGNI，各牌堆行为差异大 |
| Survivor 牌库 | 独立 SurvivorDeck 类 | Survivor 内嵌字段（deck/discard_pile） | ponytail，避免空壳类 |
| 事件卡实现 | 单设 EventCardData 子类 | ActionCardData + forced=true TRIGGER | 不增加新子类，沿用 ECA 模式 |
| 怪物天赋实现 | 部分用 PASSIVE | 均用 forced=true TRIGGER | 统一锁定技表达 |
| 任务物品卡实现 | 与装备卡共用字段 | 独立 ItemCardData（与 EquipmentCardData 平级） | 任务物品无 size/max_ammo 等装备字段 |
| 燃料/弹药字段 | 分离 max_fuel / max_ammo | 复用 max_ammo/current_ammo + ammo_type | ponytail，避免字段重复 |
| 装备卡技能管理 | 通过 Entity.skill_ids | SkillExecutor 单独遍历 equipment_zone | 装备卡进出频繁，技能列表动态变化 |
| GameSession | autoload 单例 | 场景根节点（每局新建） | 避免状态残留，无需 reset 逻辑 |
| 骰子机制 | 内嵌各处 | DiceRoller 独立静态工具类 | 复用 + 无状态 |
